策略模式和工厂模式的应用

减少if-else的使用

首先，定义一个策略接口，表示不同的价格计算策略：

public interface Strategy {
    void apply();
}

然后，实现不同的策略类，每个类负责实现一种具体的价格计算策略：

// 实现具体的策略
public class ConditionOneStrategy implements Strategy {
    @Override
    public void apply() {
        // 实现价格计算逻辑
        System.out.println("支付满15减1");
    }
}

// 实现具体的策略
public class ConditionTWOStrategy implements Strategy {
    @Override
    public void apply() {
        // 实现价格计算逻辑
        System.out.println("支付满18减2");
    }
}


// 其他策略类...

接下来，创建一个工厂类，用于创建对应的策略类对象：

// 创建策略工厂
public class StrategyConditionFactory {
    // 保存条件和对应的策略
    private Map<String, Strategy> conditions = new HashMap<>();

    /**
     * 构造方法，初始化条件和对应的策略
     */
    public StrategyConditionFactory() {
        conditions.put("ConditionOne", new ConditionOneStrategy());
        conditions.put("ConditionTwo", new ConditionTwoStrategy());
        // 添加更多条件和对应的策略
    }
    /**
     * 根据条件获取对应的策略对象
     * @param condition 条件
     * @return Optional包装的策略对象，如果条件不存在对应的策略，则返回空的Optional
     */
    public Optional<Strategy> getStrategy(String condition) {
        return Optional.ofNullable(conditions.get(condition));
    }
}
  • 最后，在主代码中使用策略模式和工厂模式来计算价格：
// 使用策略模式和工厂模式
public class YourClass {
    // 创建一个私有的不可变的 StrategyConditionFactory 对象，用于生成策略条件
    private final StrategyConditionFactory strategyConditionFactory = new StrategyConditionFactory();

    /**
     * 根据给定的条件决定选择相应的策略进行执行
     * @param someCondition 条件字符串
     */
    public void decide(String someCondition) {
        Strategy strategy = strategyConditionFactory.getStrategy(someCondition)
                .orElseThrow(() -> new IllegalArgumentException("Wrong condition"));
        strategy.apply();
    }
    public static void main(String[] args) {
        YourClass yourClass = new YourClass();
        yourClass.decide("ConditionOne");
        yourClass.decide("ConditionTwo");
        yourClass.decide("ConditionThree");
    }
}

通过使用策略模式和工厂模式，您可以将不同的价格计算逻辑封装到不同的策略类中，避免了if-else语句的冗长和复杂。同时，通过工厂模式，您可以根据具体的类型选择和创建对应的策略类，提高了代码的可扩展性和可维护性。
